Automated Red-Teaming for AI Agents

Crimson is a multi-stage adversarial assessment platform that discovers and exploits vulnerabilities in LLM-based agent systems. It uses a four-stage pipeline — recon, plan, attack, and report — to systematically probe target agents for prompt injection, data exfiltration, tool misuse, and policy bypass weaknesses.

/// Attack Categories

🔍

SYSTEM_PROMPT_EXFIL

Extract hidden system prompts and internal instructions from the target agent.

👤

PII_EXFIL

Exfiltrate personally identifiable information from agent memory or data stores.

🔑

SECRET_EXFIL

Leak API keys, credentials, and other secrets through crafted prompts.

🔧

TOOL_MISUSE

Trick the agent into calling tools with malicious or unintended parameters.

🚫

POLICY_BYPASS

Circumvent safety guardrails and content policies through adversarial inputs.

🌐

CROSS_TENANT

Access data or actions belonging to other users or tenants via injection.

/// How It Works

01

RECON

Introspect target agent source code, system prompts, tool definitions, and data stores to map the full architecture.

02

PLAN

Analyze the architecture graph for attack surfaces and data flows. Generate a prioritized attack plan with tailored strategies.

03

ATTACK

Execute multi-turn adversarial conversations against the target. Each attack probes a specific vulnerability with adaptive tactics.

04

REPORT

Aggregate findings, calculate risk scores, and generate a comprehensive hardening report with actionable remediations.
